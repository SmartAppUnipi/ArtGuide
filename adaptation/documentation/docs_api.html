Follows a detailed explanation of the json files required and provided by the Adaptation team.



This is a template of the input_phase1.json file that Adaptation team expects to receive from the Smart Deep AI 4 Search (SDAIS) team. More in detail, among the information flowing from the Classification module, the Adaptation module expects to find an array of users's tastes and a language tag, as shown aside.


On the right, an example of output_phase1.json, which the Adaptation module sends through a "POST" request to the SDAIS module as a response to input_phase1.json.
The structure of such file is the following:
  • keywordExpansion is a list of objects
  • Each of such objects represents the semantic augmentation of the meaning of on of the uTastes received through the input_phase1.json and it is represented as a list of keywords.

Here an example of what the Adaptation module expects from SDAIS module as an answer to the output_phase1.json "POST" request, namely the most promising results to the personalized user query.
The results object is a list of "answer" objects, each of the ones, characterized by the following fields:
  • A list of keywords, those the SDAIS module used to perform the query that led to the retrieval of
  • url field
  • A title field of the web page (scraped by SDAIS)
  • The sections present in the text as objects (title and content)

The Adaptation module provides to SDAIS as final output a json file which contains, in addition to the information already present in the previous messages, the tailored text (tailoredText), which sould be displayed to the user.
